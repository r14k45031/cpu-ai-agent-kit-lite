# 使用說明（詳細操作手冊）

這份手冊假設你已經照 README.md 執行完 `install.bat`。
以下所有操作都不需要網路。

完整版的主力模型是 `qwen3-30b`（需 32GB 記憶體）；
輕量版或記憶體較小的電腦請把文中的模型一律換成 `qwen3-4b`。

---

## 一、系統組成（先建立概念）

| 元件 | 角色 | 你會怎麼接觸它 |
|---|---|---|
| Ollama | 推論引擎，在背景執行模型 | 幾乎不用管它，開機自動啟動 |
| AnythingLLM | 對話視窗＋文件知識庫 | 日常主要使用的程式 |
| tools 資料夾 | 批次處理工具 | 把檔案拖到 .bat 上 |
| pandoc / Tesseract / Python | 底層工具 | 不會直接接觸 |

資料流：你的文件 → AnythingLLM 或 tools → 本機 Ollama 模型 → 結果。
**全程不出這台電腦。**

---

## 二、AnythingLLM 第一次設定（一步一步）

1. 開始選單開啟 **AnythingLLM**，點 **Get started**。
2. **選擇 LLM**：清單中找到 **Ollama**（不是 OpenAI！）。
   - Ollama Base URL：`http://127.0.0.1:11434`（通常會自動偵測）
   - Chat Model：選 `qwen3-30b`（或 `qwen3-4b`）
   - Token context window：填 `8192`
3. 後續的資料處理問卷可以直接 Skip。
4. 建立第一個**工作區（Workspace）**，名稱隨意，例如「合約文件」。
5. 設定嵌入模型（讓知識庫看得懂中文的關鍵）：
   - 點左下角**扳手圖示**（Settings）→ **AI Providers** → **Embedder**
   - 選 **Ollama**，模型選 `bge-m3`，按 Save
6. （建議）Settings → **Customization** → Display Language 改成**中文**。
7. 向量資料庫用預設的 LanceDB 即可，不用改。

> 介面版本不同，選項名稱可能略有差異，但流程相同。

### 把文件放進知識庫

1. 進入工作區，點對話框旁的**上傳圖示**（迴紋針／上傳箭頭）。
2. 把 PDF、Word、txt 拖進視窗。
3. 勾選檔案 → **Move to Workspace** → **Save and Embed**。
4. 等進度跑完（CPU 嵌入較慢，一份 50 頁的 PDF 約 1～3 分鐘）。

### 兩種對話模式（很重要）

工作區設定（齒輪）→ Chat Settings → Chat mode：

- **Chat**：一般對話，模型可以自由發揮，也會參考文件。
- **Query**：**只**根據知識庫裡的文件回答，答不出來會直說。
  查合約條款、規章內容時用這個，可大幅減少模型瞎編。

如果某份文件特別重要，在文件清單裡把它 **Pin**（圖釘）起來，
整份內容都會進入上下文，回答會更準確（但更耗記憶體）。

### 在對話中看圖（視覺模型）

想在 AnythingLLM 裡直接問圖片：另建一個工作區，
在工作區設定 → Chat Settings 把 Chat model 改成 `qwen3-vl`，
之後用對話框的附件功能附上圖片提問即可。
（或者更簡單：直接用 tools 的「拖入圖片AI解讀.bat」。）

## 三、日常文書工作的提示詞範例

直接複製後修改使用：

**摘要**
```
請將這份文件摘要成一頁：先 3 句話總覽，再條列 8 個以內的重點，
最後列出需要注意的日期與金額。
```

**潤稿**
```
請潤飾以下文字：語氣專業、去除贅詞、修正錯字，保留原意，
輸出改好的版本即可，不要解釋改了什麼。

（貼上你的文字）
```

**翻譯**
```
請將以下英文翻成繁體中文（台灣用語），專有名詞附上原文括號：

（貼上原文）
```

**草擬公文／信件**
```
請幫我草擬一封正式信件：
- 收件對象：（例：合作廠商）
- 目的：（例：通知交貨延遲兩週並致歉）
- 語氣：正式但不僵硬
- 長度：300 字內
```

**萃取結構化資料**
```
請從這份文件中整理出所有「品項、數量、單價、交期」，用表格輸出。
```

## 四、tools 拖放工具詳解

> 所有工具都有兩種叫法：把檔案**拖到 .bat 圖示上**，或對檔案
> **按右鍵 → 傳送到 → 「AI ○○」**（安裝時已自動加入右鍵選單）。

### 拖入文件做摘要.bat

- **拖一個檔案**：產生 `原檔名_摘要.docx` 在同一個資料夾。
- **拖整個資料夾**：資料夾內每份 PDF/Word/PowerPoint/Excel/txt 各自產生一份摘要，
  適合一次處理幾十份文件（放著讓它跑即可）。
- 遇到沒有文字層的掃描 PDF 會自動先 OCR 再摘要；圖片檔（PNG/JPG）也能直接拖入。
- 超長文件會自動分段摘要後再整合。
- 範例：拖入 `年度報告.pdf` → 得到 `年度報告_摘要.docx`。

### 拖入PDF或圖片做OCR.bat

- 給「掃描的」PDF 用：整頁是影像、無法選取文字的那種。
- 也接受單張圖片：PNG、JPG、TIF、BMP——文件拍照、螢幕截圖都可以。
- 輸出 `_OCR.txt`（純文字）和 `_OCR.docx`（Word）。
- 速度約每頁 3～10 秒；直式排版和低解析度掃描的辨識率較差。
- OCR 完的文字檔可以再拖去做摘要。

### 拖入Word改寫.bat

- 拖入 .docx / .txt / .md，會跳出視窗問你**改寫指示**：
  - 直接按 Enter ＝ 預設潤稿
  - 或輸入自訂指示，例如：`改成新聞稿風格`、`縮短成一半長度`、
    `把口語改成正式書面語`
- 輸出 `原檔名_改寫.docx`。
- 注意：輸出是**新檔案**，原檔的字型、表格、圖片不會保留。

### 拖入圖片AI解讀.bat

- 用**視覺模型**（qwen3-vl）真正「看懂」圖片，和 OCR 只讀文字不同：
  可以描述照片、解讀圖表趨勢、辨認手寫註記、回答關於圖片的問題。
- 拖入圖片後會問你想問什麼：
  - 直接按 Enter ＝ 詳細描述內容＋列出圖中文字
  - 或輸入問題，例如：`這張圖表哪一年成長最快？`、`這張發票的總金額是多少？`
- 輸出 `原檔名_解讀.docx`，同時顯示在畫面上。
- 什麼時候用 OCR、什麼時候用這個？**只要文字**（要拿去編輯、搜尋）用 OCR，
  精準度較高；**要理解內容**（圖表、照片、版面複雜的文件）用這個。

### 拖入Word就地AI改寫.bat（深度編輯）

- 和「拖入Word改寫」的差別：這個工具**逐段改寫後寫回原本的版面**——
  段落樣式、標題階層、表格結構、圖片、頁首頁尾全部保留。
- 拖入 .docx 後輸入指示（Enter＝潤稿），輸出 `原檔名_就地改寫.docx`，原檔不動。
- 安全機制：太短的段落（標題）和數據格（金額、日期）自動跳過不改，
  提示詞強制要求數字原封不動。
- 限制：被改寫段落「內部」的字元級格式（句中幾個字加粗）會統一。

### 拖入Excel分析.bat

- 拖入 .xlsx / .csv，輸入你想問的問題，例如：`哪個產品成長最快？`
  直接按 Enter ＝ 自動產出完整分析報告（概況、關鍵數字、趨勢異常、建議）。
- 數值欄位的總和、平均、最大最小由**程式精確計算**後才交給 AI 解讀，
  數字不會被模型算錯。
- 輸出 `原檔名_分析.docx`。

### 拖入Excel欄位AI處理.bat（寫入型）

- AI **逐列**處理資料，把結果寫進新欄位。例如拖入客訴清單，
  指示「根據備註欄判斷急件或一般件」，新欄位名稱填「分類」。
- 適合的任務：**分類、翻譯、摘要、萃取關鍵字**這類文字判斷。
  數值大小比較請直接用 Excel 公式（小模型比大小偶爾會錯，公式永遠是對的）；
  若必須用 AI 做數值判斷，請在 32GB 記憶體的電腦上讓它用 qwen3-30b。
- 輸出 `原檔名_AI處理.xlsx`（原檔不動；其他工作表保留，圖表圖片不會帶過去）。
- 速度：每 5 列一批送模型，300 列以上會提醒預估時間。

### 拖入文件變簡報.bat

- 拖入 Word / PDF / txt / md，AI 自動整理成 6～12 張的簡報大綱，
  直接產生 PowerPoint 檔（`原檔名_簡報.pptx`）。
- 產出的是乾淨的白底簡報，開啟後在 PowerPoint 套用任何佈景主題即可美化。
- 適合：報告轉簡報、會議資料快速成型。

### 拖入MD轉Word.bat

- AI 產出的 Markdown 文字（含 # 標題、- 條列、表格）存成 .md 檔後拖入，
  轉出排版乾淨的同名 .docx。
- 搭配 AnythingLLM 使用：請模型「用 Markdown 格式輸出」，
  把回答存成 .md，拖進來就變 Word。

### 啟動資料夾監看.bat（全自動批次）

- 雙擊啟動後會出現一個監看視窗（**關閉視窗即停止**），並自動建立
  `文件\AI自動處理\` 資料夾，內含「收件匣」「完成」「失敗」。
- 把檔案丟進**收件匣**就自動處理：文件（PDF/Word/PPT/Excel/txt）自動摘要、
  圖片自動 OCR。處理完原檔＋結果一起移到**完成**，出錯的移到**失敗**。
- 所有動作記錄在 `處理紀錄.txt`。
- 適合：每天固定要處理一批文件的流程——早上把檔案全丟進去，去做別的事。
- 想開機自動啟動：對「啟動資料夾監看.bat」按右鍵建立捷徑，
  把捷徑放進「執行」輸入 `shell:startup` 開啟的資料夾。

## 五、不開圖形介面的用法（進階）

命令提示字元直接對話：

```
%LOCALAPPDATA%\Programs\Ollama\ollama.exe run qwen3-30b
```

（輸入 `/bye` 離開。）

寫自己的自動化：Ollama 在 `http://127.0.0.1:11434` 提供 OpenAI 相容 API，
`tools\scripts\ai_common.py` 就是現成的呼叫範例，可以直接改。

## 六、效能與品質調校

- **回答速度慢**：純 CPU 的正常表現（30B 約每秒 5～15 字）。
  跑批次摘要時把電腦放著讓它做完即可；互動使用嫌慢就換 `qwen3-4b`。
- **第一句等很久**：模型從硬碟載入記憶體需要時間，第一次最慢，之後就快了。
  Ollama 預設模型閒置 5 分鐘後卸載，久沒用再問會再等一次。
- **記憶體吃緊**：關掉瀏覽器大量分頁再跑 30B；或改用 4B。
- **一次能讀多長**：上下文設定為 8192 tokens（約 6000～8000 中文字）。
  更長的文件請靠知識庫檢索（RAG）或 tools 的自動分段，不要整篇貼進對話。
- **回答品質差**：先確認用的是 30B 不是 4B；查文件內容改用 Query 模式；
  重要文件用 Pin。

## 七、疑難排解

| 症狀 | 處理 |
|---|---|
| 「連不上 Ollama」 | 開始選單開一次 Ollama；或重新開機 |
| AnythingLLM 找不到模型 | Settings → AI Providers → LLM 重新選一次 Ollama |
| 知識庫查不到中文內容 | 確認 Embedder 是 `bge-m3`（不是內建預設值），改完要**重新上傳文件** |
| OCR 全是亂碼 | 原稿解析度太低，或不是繁中／英文文件 |
| 拖放工具閃退 | 開命令提示字元手動執行看錯誤訊息：`python tools\scripts\summarize.py 檔案路徑` |
| 模型一直輸出簡體字 | 在提示詞加一句「請一律使用繁體中文（台灣用語）」 |

## 八、資料都存在哪（備份與隱私）

- 模型：`C:\Users\<使用者>\.ollama\models`
- AnythingLLM 知識庫與對話：`C:\Users\<使用者>\AppData\Roaming\anythingllm-desktop`
- 換電腦時備份以上兩個資料夾即可。
- 這套系統完全離線運作，你的文件不會離開這台電腦。
